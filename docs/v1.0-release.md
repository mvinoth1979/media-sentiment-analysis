## Supabase Schema State (v1.0)

Key tables active at v1.0:

```sql
-- Core tables
articles          -- all ingested content (news + YouTube)
brands            -- brand registry with keywords/languages/YouTube config
user_roles        -- RBAC assignments (user_id, role, brand_id, agency_id)
agencies          -- agency tenants
annotations       -- sentiment trend chart annotations (date, label, brand_id)
alert_configs     -- email alert thresholds per brand

-- Key columns on articles
id, brand_id, portal_id, source_type, title, url, content,
sentiment_label, sentiment_score, language, author,
topics[], entities[], states_mentioned[], keywords[],
source_credibility, content_hash, reach_metadata (jsonb),
collected_at, published_at, model_used
```

To restore schema from a backup run:
```powershell
# If you have a Supabase schema dump saved:
# psql $SUPABASE_DB_URL < scripts/supabase-schema-v1.0.sql
# (See scripts/dump-supabase-schema.ps1 to generate a fresh dump)
```

## How to Roll Back

### Quick command (recommended)
```powershell
# Open PowerShell in project root
.\scripts\rollback-v1.0.ps1
```

### Manual steps (if script fails)
```powershell
# 1. Restore all files to v1.0 state
git fetch --tags origin
git checkout v1.0 -- .

# 2. Commit and push (triggers Railway + Vercel auto-redeploy)
git add -A
git commit -m "rollback: revert to v1.0"
git push origin master
```

### Railway-only rollback (backend only)
1. Go to https://railway.app → your project
2. Click **Deployments** tab
3. Find the deployment for commit `06f0f04`
4. Click **•••** → **Redeploy**

### Vercel-only rollback (frontend only)
```powershell
# Option A: CLI
cd frontend
vercel --prod

# Option B: Dashboard
# https://vercel.com → your project → Deployments
# Find the deployment for commit 06f0f04 → Promote to Production
```
